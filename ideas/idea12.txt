“You chose C because it seems to match the surface wording, but it fails when the question asks for the result after the second stage, not the first.”

5. Risk trigger rules
A premium system should not flag risks arbitrarily. Each risk must have a trigger logic.

Flag when intervention is active but expected movement does not occur within the review window.

6. Intervention lifecycle
Now define how interventions behave from start to finish.

An intervention is not just “assign more questions.”
It is a managed treatment cycle.

Intervention states
1. Detection
The system identifies a meaningful issue.

2. Qualification
The system checks whether the issue is real, repeated, severe enough, and actionable.

3. Assignment
The issue is mapped to an intervention class.

4. Activation
The child begins a targeted cycle.

5. Monitoring
The system tracks whether the intervention is working.

6. Review
The system evaluates evidence at the review point.

7. Resolution, continuation, redesign, or escalation
Based on outcome.

Example intervention lifecycle
Problem: timed science reasoning weakness

Detection
Child performs adequately in untimed science but repeatedly underperforms in timed interpretation.

Qualification
Pattern confirmed across 3 sessions and 2 question families.

Assignment
Intervention class: pressure adaptation + reasoning repair

Activation
System schedules short timed sets with guided review

Monitoring
Track hesitation time, completion rate, accuracy under time

Review point
After 6 days or 4 intervention sessions

Possible outcomes

resolved: timed band improved sufficiently

continue: signs of progress but not enough

redesign: speed improved but reasoning remains weak

escalate: persistent failure despite active intervention

This makes the intervention engine feel clinical and premium.

13. Risk Trigger Rules
Each risk should have explicit detection logic.

Memory decay
Trigger when a topic previously classified as stable falls below retention threshold across two spaced checks.

Slow completion
Trigger when untimed performance remains strong but timed performance repeatedly falls below target pace.

Misconception recurrence
Trigger when the same reasoning error occurs across multiple items within the same question family.

Plateau
Trigger when an intervention has been active through its review window but movement remains below expected improvement threshold.

Confidence instability
Trigger when performance drops materially after one or two wrong answers within the same session.

Avoidance
Trigger when the child repeatedly under-engages with a high-priority subject or defers it across multiple cycles.

14. Strategy Engine: Decision Logic
The strategy engine decides what the child should focus on now and how that changes over time.

14.1 Inputs

14.2 Outputs

topic priority ranking

escalation recommendation

14.3 Core decision principles
Principle 1: Stop mark leakage first
The system should prioritize issues most likely to cause near-term lost marks.

Principle 2: Stabilize before accelerating
If knowledge is fragile, do not intensify speed too early.

Principle 3: Protect memory before decay spreads
Timely reinforcement is better than later repair.

Principle 4: Do not over-focus on already stable areas
Premium strategy means disciplined allocation.

Principle 5: Shift visibly when evidence changes
Parents should see real strategy adaptation.
