The general label is commonly confused terms or confusable terms.

In teaching, assessment, or glossary design, I would group them under one of these names:

Commonly confused terms — the clearest everyday label.

Confusable pairs — useful when they come in twos.

Contrast terms or contrast pairs — useful when you want to teach the difference explicitly.

Near-neighbor concepts — useful in a more technical learning-design sense.

With your examples:

sepals vs petals = a commonly confused pair within related plant structures.

isotopes vs isomers would also be a commonly confused pair in chemistry. Isotopes are variants of the same element with different neutron counts, while isomers have the same molecular formula but different structures. 

For your second example, you wrote “isomorphes.” If you meant isomorphs / isomorphism, that usually refers to substances with similar crystal structure or form, which is a different concept again. 

For your app, the best naming would be:

Commonly Confused Terms
or, if you want it to sound smarter,
Concept Contrast Pairs

That second one is especially good if the goal is not just definition, but helping students separate similar ideas clearly.

2. Similarity Trap
What it is
A deception mode that focuses on where the two concepts overlap and where learners get fooled.

Difference Drill asks:
What belongs to which?

Similarity Trap asks:
What makes them seem alike, and where does that similarity mislead you?

Core purpose
To expose false confidence.

This is where many students who think they know the answer get caught.

Best screen structure
Center:

one statement card at a time

Below:
multiple response logic such as:

true for both

true for A only

true for B only

false for both

Or:

overlap

A only

B only

trap

You can theme these beautifully.

Example with osmosis vs diffusion
Statement:
“Both involve movement from higher concentration to lower concentration.”

Answer options:

true for both

osmosis only

diffusion only

false for both

Another:
“Both require a selectively permeable membrane.”

Correct:

osmosis only

Another:
“Both can occur in living cells.”

Correct:

true for both

Another:
“Perfume spreading in air is osmosis.”

Correct:

false / trap

Better interaction mechanic
Similarity Trap should feel more dangerous than Difference Drill.

Possible presentation:

the statement appears in a glowing center card

the learner has a few seconds to classify it

trap phrases are deliberately worded to sound convincing

some cards are “double traps” where part of the statement is true but the conclusion is false

Trap card types
partial truth cards

overlap cards

false equivalence cards

disguised example cards

keyword trap cards

misleading diagram cards

half-correct statement cards

Special replay style
After each question, the review should explain:

what part of the statement was true

what part made it deceptive

what clue should have warned the learner

Example:
Why this was a trap:
Both processes involve concentration ideas, but only osmosis specifically concerns water moving across a selectively permeable membrane.

What it diagnoses
overlap confusion

sloppy reading

keyword dependence

poor precision

incomplete conceptual boundaries

Unique value
This mode teaches:
“Just because two ideas are related doesn’t mean they are interchangeable.”

That is a very important academic skill.

Describe the logic behind which is which 

Thought for 12s
Which Is Which is the fluency engine in the Traps family.

Where Difference Drill checks whether a learner can sort evidence carefully, Which Is Which checks whether the distinction has become instant and automatic. Its logic is built around one question:

Can the student recognize the correct concept fast enough that it feels obvious?

Core purpose
It is not mainly trying to teach from scratch.
It is trying to test and strengthen:

instant recognition

reaction speed

confidence

precision under pressure

whether the concept boundary is now internalized

A learner may understand a pair in Know the Difference, and may even do fairly well in Difference Drill, but still hesitate too much.
That hesitation means the concept is not yet fluent.

That is what Which Is Which is for.

The basic logic
The system takes a pair, for example:

Osmosis vs Diffusion

Then it pulls short, decisive clues from the pair’s contrast profile.

Examples:

“Water only”

“Perfume spreading”

“Membrane required”

“Ink in water”

“Root hair cell uptake”

For each clue, the learner has to choose immediately:

Osmosis or Diffusion

So the logic is:

show a short clue

force a fast binary choice

measure correctness

measure reaction time

detect hesitation patterns

adapt difficulty or speed

review missed items afterward

Why the prompts must be short
This mode is not for long reading.

If the prompt is long, then the mode starts behaving like Difference Drill.
That would defeat the point.

So Which Is Which should use:

short phrase fragments

keyword clues

mini examples

diagram flashes

tiny scenario snippets

The learner should be able to answer in 1–3 seconds if they really know it.

That is the design logic.

What counts as a valid prompt
A prompt in Which Is Which must do one of two things:

It must either:

A. strongly point to one concept, or
B. reveal whether the learner still hesitates on a concept they should already know

Good examples:

“Selective membrane”

“Any particles”

“Water molecules only”

“Spreads in air”

“Across membrane”

Bad examples:

long explanatory sentences

vague overlap statements

prompts that require deep reasoning

prompts that belong equally to both unless you are intentionally testing advanced fluency breakdown

So the backend logic should filter prompts for speed suitability.

How the engine decides what to show
The mode should pull from the pair’s contrast profile, especially from these lanes:

keyword clue

short definition fragment

characteristic fragment

condition fragment

example fragment

mini diagram

application fragment

It should favor decisive clues first, then move toward more subtle ones.

So for a pair like Osmosis vs Diffusion, the internal order might be:

Easy clues
“Water only”

“No membrane required”

“Perfume spreading”

Medium clues
“Root hair cell”

“Particles spread from high concentration”

“Across a selectively permeable membrane”

Hard clues
disguised mini-scenarios

cropped diagrams

partial phrases

clues that are close but still distinguishable

That is how the mode progresses from obvious to refined.

What the mode is actually measuring
It is measuring more than right or wrong.

It should track at least four things:

1. Accuracy
Did the learner choose the right concept?

2. Reaction speed
How quickly did they answer?

3. Hesitation
Did they pause too long even when correct?

4. Consistency
Do they repeatedly fail the same clue type?

This matters because these are different situations:

fast + correct = fluent mastery

slow + correct = understood, but not automatic yet

fast + wrong = impulsive confusion

slow + wrong = weak distinction

That is the hidden intelligence behind the mode.

The scoring logic
Which Is Which should not be scored only by accuracy.

It should use a blended score.

For example:

Fluency Score = Accuracy + Speed + Consistency

A learner who gets 9/10 correct but takes too long may still have lower fluency than someone who gets 8/10 quickly and steadily.

So the mode can produce outputs like:

Recognition accuracy: 90%

Average response time: 2.8s

Hesitation rate: 40%

Fluency level: Developing

That is much more meaningful than just “You scored 9/10.”

How timing should work
Each prompt should have a short time window.

For example:

easy level: 4 seconds

standard level: 3 seconds

pressure level: 2 seconds

elite level: 1–1.5 seconds

If the learner does not respond:

it can count as skipped or missed

it should be logged as hesitation

it should appear in review

That is important because timeout itself is diagnostic.

If a student repeatedly times out on membrane-related clues, the issue may not be total ignorance. It may be uncertain recognition.

How the sequence should behave
Which Is Which should feel fast and rhythmic.

The flow should be:

prompt appears → learner taps → immediate feedback → next prompt

The system should avoid heavy interruption between prompts.

Correct answer:

tiny success pulse

streak increases

next prompt loads quickly

Wrong answer:

quick correction signal

correct answer flashes briefly

next prompt continues

The rhythm is part of the learning.
The learner builds recognition through repeated exposure.

How diagrams fit in
Which Is Which becomes much stronger if it includes mini diagram flashes.

Instead of only words, the system can flash:

a membrane diagram

arrows crossing a cell boundary

particles spreading in open space

Then ask:
Osmosis or Diffusion?

That tests visual fluency, not just verbal fluency.

This is very important because many students memorize words but fail to recognize the same concept in a figure.

How it differs from Difference Drill
This distinction is crucial.

Difference Drill
Asks:
Where does this belong?

It is slower, more visible, more diagnostic, more stack-based.

Which Is Which
Asks:
Quick — what is this?

It is faster, leaner, more reflexive, more fluency-based.

Difference Drill tests sorting.
Which Is Which tests automatic recognition.

They are related, but they are not the same.

Adaptive logic
This mode should adapt based on performance.

If the learner is doing well:

shorten timer

increase pace

use subtler clues

add mini diagrams

mix in near-traps

If the learner is struggling:

return to stronger clues

slow the pace slightly

repeat the most confused lanes

focus on the clue types causing hesitation

So the engine should not just ask random prompts.
It should learn from:

wrong answers

slow answers

repeated lane failures

For example, if the learner is always slow on application clues but fine on keyword clues, the mode can keep targeting application-based recognition.

End-of-round logic
At the end, the system should not only say what was wrong.

It should say what kind of fluency the learner has.

Example:

You recognize definitions quickly.

You hesitate on examples.

You are weakest on diagram-based clues.

Your confusion rises when the clue is indirect.

That turns the mode into a real intelligence feature.

Review logic
Which Is Which review should be lighter than Difference Drill review.

Because this mode is about speed, the replay should focus on:

what you missed

what you hesitated on

what clue should have made it obvious

Example:

Prompt: “Water only”
You chose: Diffusion
Correct: Osmosis
Missed clue: “water only” is the strongest identity marker for osmosis.

If the learner answered correctly but too slowly, the review can still say:

You got this right, but you took too long.
That means the concept is understood but not yet automatic.

That is very powerful.

The true educational value
Which Is Which is useful because exams often reward more than understanding.
They reward recognition speed.

A student under pressure must be able to see a clue and identify the right concept quickly.

So this mode helps convert:

knowledge → fluency

fluency → confidence

confidence → performance under pressure

That is the real logic behind it.
