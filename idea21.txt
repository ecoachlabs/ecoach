Performance value

4. Ambiguity detection
Sometimes the DB will return multiple plausible directions.

Example:
A learner says, “I struggle with equations.”

That could mean:

balancing chemical equations

solving algebraic equations

word-problem translation

formula substitution

simultaneous equations

At that point, the coach should not guess too hard.

It should detect ambiguity and ask the learner a targeted question such as:

Are you struggling with solving them, balancing them, or understanding what they mean?

Does this happen more in math or science?

Do these examples match your problem?

This is exactly where the user confirmation loop belongs.

Not everywhere. Only where it meaningfully improves accuracy.

5. Applicability confirmation
This is one of the smartest features you mentioned.

When the coach finds candidate patterns or interventions, it can return a short confirmation set like:

Do these apply to you?

Which of these sounds most like your challenge?

Is your problem more about speed, confusion, forgetting, or making careless mistakes?

This lets the coach refine the next search and avoid misdiagnosing the learner.

So the coach is allowed to say, in effect:

“I found three likely patterns in your case. Help me confirm which one fits you best so I can tailor the right intervention.”

That is excellent design.

6. Missing-piece generation

no level-appropriate recap card

concept graph

known question archetypes

learner weakness data

That way generation is grounded, not random.

The three search lenses it should use
Your coach should search through three lenses at the same time.

1. Curriculum lens
“What is officially relevant here?”

This keeps it aligned to level, syllabus, and exam reality.

2. Pedagogical lens
“What kind of resource is needed for this objective?”

This decides whether the learner needs:

explanation

worked example

drill

comparison

recap

challenge

misconception repair

3. Learner lens
“What fits this learner specifically?”

This considers:

weak areas

preferred mode

recent mistakes

pressure sensitivity

confidence state

speed vs accuracy issue

When these three lenses overlap, the search becomes powerful.

The decision engine you need
I would formalize this as a Coach Academic Decision Loop.

Stage A: Determine intent
What is the coach trying to achieve?

Stage B: Locate syllabus zone
What curriculum node governs this problem?

Stage C: Scan resource graph
What relevant material exists in the DB?

Stage D: Score candidates
What best fits this objective and learner state?

Stage E: Resolve ambiguity
Is the best interpretation clear enough?
If not, ask the learner a short applicability question.

Stage F: Compose session
Build the best flow from found resources.

Stage G: Generate missing parts
Fill the gaps intelligently.

Stage H: Deliver and observe
Watch performance.

Stage I: Update memory
Store what was learned.

That is the loop.

The syllabus should do more than filtering
The syllabus is not just a tag or category. It should actively shape search and generation.

It should help the coach answer:

what concepts are mandatory here

what depth is correct for this class

what skills should be tested

what common exam forms apply

what prerequisites should be reinforced

what sequence of teaching makes sense

So the syllabus acts as a constraint system and a search optimizer.

Without this, the coach may retrieve academically valid content that is still wrong for the learner’s level.

ask for confirmation first

So the coach should ask only when ambiguity is real and meaningful.
