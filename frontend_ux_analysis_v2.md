After exhaustive line-by-line extraction of all 38 idea documents, the eCoach product vision contains **2,183 distinct features, sub-features, screens, components, interactions, engine outputs, data visualizations, animations, sounds, notifications, reports, settings, and admin tools**.

This is not a test-prep app. This is an **academic intelligence operating system** comparable in scope to a full ERP or enterprise platform. The frontend alone demands 200+ unique screen types, 80+ reusable components, and 3 distinct UI shells.

## COMPLETE FEATURE INVENTORY BY DOMAIN

### A. STUDENT LEARNING MODES (Major Feature Surfaces)

1. **Journey Mode** -- Goal-based adaptive mastery with 5 phases, milestone exams, stations, mastery ladders, route modes (steady/intense/recovery/custom/rescue), knowledge heat map with 7 dimensions, 12+ question intents, misconception-aware distractors, linked question sets, and emotional intelligence layer
2. **Beat Yesterday / Climb Mode** -- Daily micro-improvement with 4-block sessions (Warm Start, Core Climb, Speed Burst, Finish Strong), 5 growth modes (Volume, Accuracy, Speed, Mixed, Recovery), 8 scoring dimensions, 6 climb states, anti-burnout detection, and teacher/parent dashboards
3. **Rise Mode / Last-to-First Engine** -- 4-stage transformation (Rescue > Stabilize > Accelerate > Dominate), deep diagnostic producing weakness maps, 8 scoring dimensions, confidence engineering, pressure training system, daily transformation sessions, teacher intervention layer
4. **Spark Mode** -- Engagement-first for disengaged students, behavioral onboarding classifying 6 student types, ultra-short missions, challenge-based framing, avatar upgrades, status titles, music themes, surprise mechanics, dynamic story mode, friend/rival mode, personality-based challenge styles
5. **Elite Mode** -- 7 performance pillars (Precision, Speed, Depth, Endurance, TrapSense, Elegant Solve, Pressure), 7 session types (Sprint, Gauntlet, Precision Lab, Depth Lab, Perfect Run, Endurance Track, Apex Mock), 6-tier progression (Foundation to Legend), Elite Performance Score formula, topic domination board, records wall, premium visual theme
6. **Knowledge Gap Mode** -- 10-layer gap taxonomy, gap scan flow, gap dashboard with 6 sections (Fix Now, Unknown, Weak but Improving, Slipping, Hidden Blockers, Recent Shrinkage), solidification sessions with 5 stages, 7 skill states, priority engine with formulas, gap ring visualization, gamification elements
7. **Memory Mode** -- 6 memory dimensions, Memory Stability Index formula, 6 session types (Scan, Rescue Burst, Deep Repair, Recall Builder, Chain Repair, Rapid Recall Drill), 6-stage recovery ladder, 5-gate confirmation sequence, 12 canonical memory states, restorative visual/sound design, memory strand animations
8. **Mock Centre** -- Ceremonial exam hall transition, distraction-free exam interface, pacing badge, confidence tagging, question flagging, multi-lens post-mock review (6 tabs), mock history/battle history, forecast insights, readiness score engine, mock types (full/topic/mini/recovery/pressure), exam hall simulation
9. **Concept Traps** -- 5 modes (Difference Drill card-sort, Similarity Trap deception, Know the Difference comparison, Which Is Which speed recognition, Unmask), physics-aware card animations, sound design, 4 difficulty levels, contrast profile data model, traps content engine
10. **Educational Games** -- Tug of War (momentum mechanics, power-ups, rope zones, comeback system), MindStack/Tetris (control-through-knowledge, morph mechanic, 7 sub-modes, boss rounds, 8 power-ups, mercy design, split-screen layout, 5-dimension difficulty model)
11. **Customised Testing Engine** -- 5-screen setup wizard (test type > subject > topics > timing > preview), 6 preparation modes (class test, midterm, mock, confidence builder, pressure, last-minute rescue), dynamic theming per mode, mid-session adaptation, readiness-based result page

## DESIGN SYSTEM REQUIREMENTS

### Token System
- **Colors**: 3 temperature palettes (student warm, parent cool, admin neutral) + 6 mode accents (Elite dark, Recovery gentle, Pressure intense, Mock serious, Game vibrant, Celebration bright)
- **Typography**: Scale optimized for readability on varying displays. Parent shell: extra-large, high-contrast. Admin: dense, compact
- **Spacing**: 4px/8px grid. Generous in parent shell, tight in admin
- **Elevation**: Card shadows, modal overlays, game HUD layers
- **Motion**: Spring-based easing, duration tokens, stagger rules, hero transitions, physics-aware animations (card-sort)
- **Sound**: Feedback SFX library, ambient modes, heartbeat/pressure sounds, celebration chimes

### Emotional Theming System (6 Modes)
| Context | Color Temp | Animation | Copy Tone | Sound |
|---------|-----------|-----------|-----------|-------|
| Normal coaching | Warm neutral | Moderate | Supportive guide | Subtle |
| Recovery/rescue | Warm amber | Slow, gentle | Protective mentor | Calming |
| Pressure/mock | Cool dark | Tight, precise | Serious examiner | Minimal |

| Game/challenge | Vibrant saturated | Fast, bouncy | Energetic challenger | Game SFX |
| Celebration | Bright warm | Expansive | Proud coach | Reward chimes |

## FINAL NOTES

- **2,183 features** is the raw count including engine outputs and backend systems. The pure frontend surface count (screens + components + interactions) is approximately **800-1,000 distinct UI elements**.
- The product is comparable in frontend complexity to Figma, Notion, or Linear -- not a typical edtech app.
- The question component alone must support 15+ input formats and is the most reused, most complex component in the system.
- Three distinct design languages (student/parent/admin) effectively triple the visual design work.
- Offline-first with local SQLite eliminates network loading states but introduces content pack management and sync UX.
- The coach voice (copy/tone/language) is as important as the visual design. A UX writing style guide with audience-specific templates is essential.
- idea7 and idea8 are identical files (confirmed duplicate).
